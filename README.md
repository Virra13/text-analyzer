# Text Analyzer

Консольное приложение на Java и Spring Boot для построения частотного словаря по материалу `.txt`-файлов указанной директории. 

## Возможности

Приложение:

* читает все `.txt`-файлы из указанной папки;
* подсчитывает количество вхождений слов;
* не учитывает регистр и знаки препинания;
* игнорирует слова короче заданной длины;
* поддерживает файл со стоп-словами;
* выводит результат в консоль или сохраняет его в JSON;
* сообщает об ошибках чтения отдельных файлов.

## Особенности обработки

* Анализируются только `.txt`-файлы, расположенные непосредственно в указанной папке. Вложенные директории не обходятся.
* Файлы читаются в кодировке UTF-8.
* Словом считается последовательность букв Unicode. Внутри слова допускаются дефис, подчёркивание и апостроф, например: `кто-то`, `don't`, `слово_слово`.
* Цифры и остальные символы не входят в состав слова.
* Регистр не учитывается: `Java`, `JAVA` и `java` считаются одним словом.
* Стоп-слова обрабатываются по тем же правилам и также не зависят от регистра.
* Слова сортируются по убыванию количества вхождений. При одинаковой частоте используется сортировка по алфавиту.
* Если отдельный файл невозможно прочитать, он пропускается, а информация об ошибке добавляется в результат.
* При отсутствии подходящих слов приложение выводит пустой результат.
* При записи в JSON родительская директория должна уже существовать. Существующий файл результата перезаписывается.
* Минимальная длина слова и количество слов в результате должны быть 1 или больше.
* Если в переданной директории нет подходящих `.txt`-файлов, приложение завершает анализ с пустым результатом.

## Структура проекта

- `cli` — обработка параметров командной строки;
- `application` — координация сценария анализа;
- `input` — чтение текстовых файлов и стоп-слов;
- `analyzer` — извлечение и подсчёт слов;
- `output` — вывод в консоль и JSON;
- `model` — модели результатов;
- `exception` — исключения приложения.
  
## Требования

- Java 17+
- Maven 3.9+
  
## Сборка

```bash
mvn clean package
```

После сборки исполняемый JAR-файл будет находиться в директории `target`.

## Запуск

```bash
java -jar target/text-analyzer-*.jar --dir=./texts --min-length=5 --top=10
```

### Параметры

| Параметр       | Описание                             | Обязательный |
| -------------- | ------------------------------------ | ------------ |
| `--dir`        | Путь к папке с `.txt`-файлами        | Да           |
| `--min-length` | Минимальная длина учитываемого слова | Да           |
| `--top`        | Количество слов в результате         | Да           |
| `--stopwords`  | Путь к файлу со стоп-словами         | Нет          |
| `--output`     | Путь к JSON-файлу с результатом      | Нет          |
| `--help`       | Показать справку                     | Нет          |

## Примеры

Вывод результата в консоль:

```bash
java -jar target/text-analyzer.jar --dir=./texts --min-length=3 --top=5
```

С использованием стоп-слов:

```bash
java -jar target/text-analyzer.jar --dir=./texts --min-length=5 --top=10 --stopwords=./stop.txt
```

С сохранением результата в JSON:

```bash
java -jar target/text-analyzer.jar --dir=./texts --min-length=5 --top=10 --stopwords=./stop.txt --output=./results.json
```

Просмотр справки:

```bash
java -jar target/text-analyzer.jar --help
```

## Пример вывода в консоль

```text

2. process - 97
3. engineering - 83
```
